## 步骤 1：在 GitHub 上建仓库（1 分钟）

1. 打开 https://github.com/new
2. 填：
   - **Repository name**: `obsidian-vault`
   - **Description**: `Obsidian Vault - second brain + multi-agent coordination (mirror of Gitee)`
   - **Visibility**: ✅ **Private**（保护 API key）
   - ❌ 不要勾 "Add a README file"
   - ❌ 不要勾 "Add .gitignore"
   - ❌ 不要选 license
3. 点 **Create repository**

## 步骤 2：在能上 GitHub 的终端 push（2 分钟）

打开 **Git Bash**（或 Windows Terminal），跑：

```bash
cd /d/ObsidianVault

# remote URL 已配好（之前 claude-desktop 加的）
git remote -v
# 应该看到：
#   github   https://github.com/haowenzhuo/obsidian-vault.git
#   origin   https://gitee.com/haowenzhuo/obsidian-vault.git

# 推 master 分支
git push github master
```

**第一次 push 会要求认证**：
- Username: `haowenzhuo`
- Password: **用你的 GitHub PAT**（不是登录密码）

如果不想每次输，配置 credential helper：

```bash
git config --global credential.helper store
# 第一次 push 后会保存 token 到 ~/.git-credentials
```

## 步骤 3：验证（30 秒）

打开 https://github.com/haowenzhuo/obsidian-vault → 应该看到所有 vault 文件

## 故障排查

### push 时 401 Bad credentials
- **检查 PAT 权限**：https://github.com/settings/tokens → 你的 token → 勾 `repo` 全选
- **检查 PAT 没过期**：可能生成时设了过期时间
- **重新生成 PAT**：https://github.com/settings/tokens/new

### push 时 Connection refused
- **网络问题**：你当前网络能不能上 GitHub？
- **代理**：你可能需要配 git proxy：
  ```bash
  git config --global http.proxy http://127.0.0.1:你的代理端口
  ```

### 仓库已存在
- 如果 GitHub 上已经有 `obsidian-vault` 仓库但不是私有的，**先在 GitHub 上删掉**再重建
