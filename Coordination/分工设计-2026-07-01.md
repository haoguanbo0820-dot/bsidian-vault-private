## 三、核心架构：任务队列 + 抢单

### 3.1 任务生命周期

```
入队（pending）→ 抢单（locked）→ 执行（running）→ 完成（done）/ 失败（failed）
```

### 3.2 4 个核心文件

```
Coordination/
├── QUEUE.json          ← 任务队列（待办 + 进行中）
├── QUEUE.suggested.json ← 建议任务（非权威 agent 提的）
├── HEARTBEAT.md         ← 所有 agent 心跳（最后活跃时间）
└── HANDOFFS/            ← 任务详情 + 上下文（点开链接看细节）
```

### 3.3 QUEUE.json 格式

```json
{
  "tasks": [
    {
      "id": "T-20260701-001",
      "title": "跑 AI 爆金日记 7/1",
      "required_skills": ["web_scrape", "python", "vault_write"],
      "priority": "high",
      "status": "pending",
      "created_by": "claude-desktop",
      "created_at": "2026-07-01T15:00:00+08:00",
      "deadline": "2026-07-01T21:00:00+08:00",
      "locked_by": null,
      "locked_at": null,
      "result": null,
      "handoff_file": "Coordination/HANDOFFS/T-20260701-001.md"
    }
  ]
}
```

### 3.4 HEARTBEAT.md 格式

```markdown
# Agent Heartbeat

| Agent | 最后心跳 | 状态 | 当前任务 | URL |
|---|---|---|---|---|
| claude-desktop | 2026-07-01 16:30 | 🟢 active | T-20260701-001 | - |
| Marvis | 2026-07-01 15:00 | 🟡 idle | - | - |
| Hermes | 2026-07-01 16:00 | 🟢 active | T-20260630-099 | 阿里云 |
| Codex | 2026-06-30 15:30 | 🔴 down | - | 主人 GUI |
| Trae CN | 2026-06-05 08:57 | ⚪ token=0 | - | 主人充值 |
```

### 3.5 抢单协议（核心）

任何 agent 启动后执行：

```python
def grab_task(my_skills):
    queue = read("QUEUE.json")
    for task in queue["tasks"]:
        if task["status"] != "pending":
            continue
        if any(skill in task["required_skills"] for skill in my_skills):
            # 抢单
            task["status"] = "locked"
            task["locked_by"] = MY_NAME
            task["locked_at"] = now()
            write_atomic("QUEUE.json", queue)  # 原子写
            if verify_lock_won(task["id"]):  # 验证抢到
                execute(task)
                return task
    return None
```

**冲突解决**：原子写 + 验证。第一个写成功的赢。

## 四、入队的 4 种来源

| 来源 | 谁能做 | 入队路径 |
|---|---|---|
| **主人交任务** | 主人 | 主人说 → 我拆 → 直接入队 `QUEUE.json` |
| **我发现可优化** | 我 | 我直接入队 `QUEUE.json` |
| **agent 建议** | Marvis/Hermes/Codex/Trae | 写 `QUEUE.suggested.json` → 我或主人审核 → 转正到 `QUEUE.json` |
| **定时触发** | 我设的 cron | 自动入队 `QUEUE.json` |

**权威等级**：
- **直接入队** = 主人 + 我（最高权威）
- **建议入队** = 其他 agent（要审核）
- **审核不通过** = 我或主人删除建议

## 七、汇报机制（触发式）

### 7.1 主动汇报的 4 种触发

1. ✅ **任务完成** → 写 LOGBOOK → 主人通知 1 次
2. ⚠️ **任何 agent 挂了/出 bug** → 立即汇报（带根因 + 修复建议）
3. ⚠️ **两个 agent 撞车** → 立即汇报（带仲裁方案）
4. ⚠️ **需要主人点 GUI**（Codex/Trae）→ 立即汇报（带具体步骤）

### 7.2 不主动汇报

- 普通 LOGBOOK 更新
- 我自己内部思考
- 凌晨 2-6 点（除非紧急）

### 7.3 主人随时可查

- 主人问"现在在干啥" → 我汇报
- `Coordination/ACTIVE.md`（实时任务）
- `Coordination/LOGBOOK.md`（历史）
