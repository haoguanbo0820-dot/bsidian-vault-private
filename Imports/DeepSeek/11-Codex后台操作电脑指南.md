### 第一步：获取 Codex 应用

2. **下载安装**

   - 或留意 Mac App Store，OpenAI 可能已上架。
   - 下载后拖入“应用程序”文件夹，首次打开需通过 macOS 的公证验证。

---

### 第二步：授予关键权限（这一步不做它就无法操控）
Codex 要在后台模拟键鼠、读取屏幕，必须获得 macOS 的两项核心权限。

1. 打开 **系统设置 → 隐私与安全性 → 辅助功能**  
   - 解锁后，把 **Codex** 开关打开。  
   - 这允许它模拟鼠标点击、键盘输入、滚动窗口。
2. 进入 **系统设置 → 隐私与安全性 → 屏幕录制**  
   - 同样把 **Codex** 打开。  
   - 这允许它“看见”屏幕内容，从而理解软件界面上的按钮、文字、数据。
3. 如有提示，再允许 **自动化** 权限（控制其他应用）和 **文件访问** 权限，以便它操作 Finder 或指定文件夹。

> 💡 注意：这些权限仅用于完成任务，OpenAI 承诺本地操作数据不会上传训练，且虚拟光标运行在隔离环境。

---

### 第三步：加载你需要的“角色插件”
Codex 强在“插件生态”，把销售、数据分析、设计等常用工具打包成一键调用的角色。

1. 在 Codex 设置界面找到 **插件市场 / 角色商店**。
2. 根据你的工作场景安装：
   - **销售角色**：连接 Salesforce、HubSpot，一句“帮我把今日线索导入 Excel 并按优先级排序”。
   - **数据分析角色**：绑定你的数据库或 Excel，会操控 Excel、Tableau 等做图。
   - **设计角色**：授权 Figma、Canva，可批量修改模板。
3. 插件会要求你进行 **账号授权**（OAuth 登录），只需按提示完成即可。

---

### 第四步：下达指令，让它“后台干活”
这才是最爽的地方——你只需用自然语言把任务完整描述出来，它就在后台自动操控软件。

**操作方式：**
- 点击菜单栏的 Codex 图标，或使用快捷键（可自定义）呼出输入框。
- 用“我要你……”开头，清晰描述：
  ```
  我要你从“销售日报.xlsx”里提取本月销售额前五的产品，
  在 PowerPoint 里做成一页图表，保存到桌面，命名为“Top5.pptx”。
  ```
- 指令下达后，你会看到后台出现一个 **半透明的小光标** 开始移动、点击、输入。此时你完全可以切回自己的窗口做别的事，两者的光标是独立的，互不争抢。

**监控与干预：**
- 你随时可以点击菜单栏图标，查看实时执行步骤（如“正在打开 Excel → 筛选数据 → 生成图表”）。
- 发现不对，直接点 **暂停** 或 **终止**，并可以修正指令，比如“不用 PowerPoint 了，换成 Google Slides”。

### 使用前必看的安全与效率建议
- **从简单任务开始**：比如“把桌面所有 PDF 重命名为‘日期_原文件名’”，逐步建立对它的信任。
- **先在不含敏感数据的文件上测试**：虽然运行于本地沙盒，初始还是建议不要一上来就处理银行账单、机密合同。

- **善用“角色”一键切换**：比如同一份数据，可先让“数据分析师”整理，再让“销售”发邮件，全程不用重新描述背景。

如果“操作软件”是指操作各种**网页版服务**（比如 Salesforce、Excel Online、Canva、浏览器里的后台系统），那么 **Operator** 是目前最好的替代。

- **它怎么工作**：OpenAI 在云端给你开一个**独立的浏览器窗口**，它在那里操控网页，完全不影响你本地的鼠标键盘。  
- **怎么用**：  
  1. 打开 Windows 上的浏览器，登录你的 ChatGPT 账号（需 Plus/Pro/Team）。  
  2. 在聊天界面里选择 **Operator** 模式（一般是一个小图标）。  
  3. 像这样下指令：“帮我登录淘宝后台，把过去 7 天的订单导出成 Excel，按地区分类。”

- **和 Codex 桌面的区别**：  
  - 它不能操控你本地的 Excel、PPT 软件，只能操控**网页**。  
  - 如果你的日常工具已经大量 SaaS 化，这基本够用。  
  - 没有插件市场，但可以自定义指令或使用 GPTs 来实现角色化。

### 路径二：用 Anthropic 的 Claude Computer Use（在 Windows 上建一个沙盒）
如果你**必须操作本地安装的桌面软件**（比如真正的 Excel 程序、财务软件），可以用 Claude 的 Computer Use 功能，自己搭建一个“沙盒里的 Windows”供它操控。

7.逆向反推文生图提示词

这个玩法，我觉得是今天最有意思的一个，不是因为它多难，而是因为它的思维方式不一样。

大多数人复刻图片的方式是：发图给 AI ，AI 猜一次提示词，生图，完事。

这个玩法不是。

它搭的是一个有测试、有反馈、有迭代的机制。

核心是两个子智能体：

A：逆向提示词智能体。 看图猜提示词。

B：监督智能体。 用 A 猜出的提示词生图，和原图找差异，把差异拆成可执行反馈，再喂回给 A。

A 猜一次，B 打一次分，A 再改。最多 5 轮，或者分数达到就停。

这很像 GAN 的逻辑，一个生成，一个判别。但这里的判别器不是数学函数，它会说人话：人物太小、文字层级没锁住、光环位置不对。每一条反馈都可以直接变成下一轮的约束。

在 Codex 里怎么用呢？

我把之前生成的权力与游戏海报图上传给 Codex ，然后粘贴这段提示词：

创建两个子智能体，一个是逆向提示词智能体 A，一个是监督智能体 B。A 的作用是猜测一张图片背后的提示词。B 智能体负责用猜出来的提示词生图，对比和原始图片的差异，然后不断给 A 反馈，让它迭代提示词。终止条件：B 认为差距很小，或者达到第 5 轮迭代。

发完之后，Codex 启动了整个迭代流程，一共跑了五轮，耗时一个多少小时。


第一轮，A 猜出大致风格：黑暗奇幻封面、火焰城堡、左侧文字区。B 生图看完说：氛围对了，但女性侧脸没抓到，徽章和文字层级都没锁住。

第二、三轮，A 陆续补进女性轮廓、纹章位置、燃烧城镇。B 每次指出新问题：女性偏右、脸朝向不对、火焰颜色太深。打分从 65% 爬到 75-80%。

1.额度翻倍的方法

这个是从卡兹克留言区发现的。

确实很实用：


首先，要知道 Codex 的额度不是按天重置，也不是按小时重置。

是一个 5 小时的滚动窗口。


你发第一条消息的那一刻，计时开始。这 5 小时内 token 用完，就只能等窗口走完才能重置。

但有一个细节很多人不知道，窗口走完之后，系统不会自动开新窗口。

它就在那等着，等你下一条消息发出去，才重新开始计算 5 小时。

所以重置时间，是你自己控制的。

举个例子。你每天下午 2 点到 6 点集中用 Codex。如果 2 点开始，窗口 7 点才重置，3 点半额度用完，你就得干等到 7 点。

但如果你上午 11 点随手给 Codex 发一句话，哪怕就说「你好」，窗口从 11 点开始，下午 4 点就重置了。

2 点开始干，4 点额度刷新，4 点接着干。

同样 4 个小时，你享受了两个窗口。

这个也可以自动设置，在左边栏找到「自动化」，新建一个，触发条件选「每天」，时间填你开始干活的 3 小时前，动作就是随便发一条消息，写啥都行。设好之后不用管，每天自动跑。

2.开启记忆功能

这个建议开，设置 → 个性化 → 记忆 → 启用记忆。


开启之后，经过一段时间沉淀，你可以在 ~/.codex/memories 目录下找到 memory.md 文件，里面是 Codex 主动记录下来的关于你的信息，你的习惯、你的项目、你的偏好。

### 路线一：零部署，直接用 Operator（最省心）

如果你的工作大部分能在**网页版软件**里完成（如 Salesforce、Excel Online、Canva、各类后台系统），那根本不需要折腾桌面。

**做法**：
1. 打开 Chrome 或 Edge，登录你的 ChatGPT 账号（需 Plus/Pro/Team）。
2. 在聊天界面选择 **Operator** 模式。
3. 用自然语言下指令，比如：“帮我登录 Notion，把本周会议纪要整理成待办清单”。

Operator 会在**云端一个独立的浏览器窗口**里干活，完全不碰你的电脑。你这边继续做自己的事，它就是那个“后台”。

> **优点**：零部署，立刻能用，完全后台。  
> **局限**：只能操作**网页**，不能操控本地安装的软件。

---

### 路线二：搭建 Claude Computer Use，让它操控一个“虚拟机里的 Windows”（稍复杂，但能操控桌面软件）

如果你必须用本地的桌面程序（如真正的 Excel、SolidWorks），那可以自己搭一个环境，让 AI 在一个**隔离的 Windows 虚拟机或 Docker 容器**里操作，不影响你主屏幕。

**大致步骤**（需要一些动手能力）：
1. 安装 **Docker Desktop**，或使用 **Windows Sandbox / VirtualBox** 创建一个虚拟机。
2. 在虚拟机里装好你需要的软件（Office、设计工具等）和 Claude Computer Use 的参考实现（Anthropic 官方提供）。
3. 通过 API 调用，让 Claude 在这个沙盒环境里操控鼠标键盘。
4. 你通过网页终端或 VNC 观察进度，文件可以共享出来。

这基本是文章里那些 macOS 效果在 Windows 上的复刻，但你要维护一个额外的“沙盒桌面”。社区已有封装好的项目如 `claude-computer-use`，跟着文档配就行。

> **优点**：能操控真正的桌面软件，实现后台隔离。  
> **局限**：技术门槛中高，需要 API 费用，且沙盒环境占用资源。

### 路线三：给 Windows 版 Codex 套一个“隔离层”（折中方案）

如果你还是想用 Codex 官方应用，可以试试让它运行在另一个 Windows 会话里，达到“后台”效果。

**做法**：
1. 在你的 Windows 里启用 **多桌面（虚拟桌面）** 功能（`Win + Tab` 新建桌面）。
2. 在 **第二个桌面** 打开 Codex，并把它需要操作的软件也放在那个桌面。
3. 当需要它干活时，切到第二桌面下达指令，然后直接切回第一桌面做自己的事。
4. Codex 在第二桌面拖动真实光标，你这边不受影响（但注意：它的鼠标点击仍在系统层，如果你在主桌面有操作，偶尔可能干扰，所以最好让它独立跑完一个任务）。

这个方法不是完美的后台，但可以做到“眼不见心不烦”。如果再结合 **Windows 多用户远程桌面** 或 **Sandbox**，就能实现真正的隔离了，不过设置会更复杂。

> **优点**：使用官方 Codex，上手快。  
> **局限**：并非真正的光标隔离，操作时尽量别在主桌面频繁点击。
