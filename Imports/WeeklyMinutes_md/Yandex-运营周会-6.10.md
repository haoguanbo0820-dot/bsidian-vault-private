**（三）订单取消/退回流程及60状态码规范**

**1. 两类取消场景操作标准**

|  |  |  |
|:---|:---|:---|
| 对比维度 | 场景一：平台/卖家取消，订单在途 | 场景二：海关拒放/我方主动拦截退回 |
| 触发条件 | 平台、卖家发起订单取消，订单已离开海关处于在途状态 | 海关判定禁止入境（危险品等）、我方主动拦截订单退回 |
| 是否抵达莫斯科仓库 | 是，正常送达 | 否，无法送达 |
| 60状态码要求 | 无需上传60状态码 | 必须上传60状态码，标注对应退回原因 |
| 俄方需提供信息 | 莫斯科仓库预计到达时间（ETA） | 60状态码+退货说明+退货物流信息 |
| 后续处理方式 | 仓库签收后分拣至退货区，统一批量退回 | 我方系统同步取消订单，无需等待货物到仓 |

**2. 统一操作规范共识**

第一，俄方回复订单查询时，禁止单一回复“订单已取消”，需明确订单流向：可抵达莫斯科仓库的，必须提供预计到仓时间；无法到仓的，需上传60状态码并详细说明取消原因及退货方向。第二，所有已取消订单均纳入中方常态化监控，未按时履约的订单将直接发起索赔流程。第三，统一行业规范关键词：Prohibited
Item（禁止进口商品）、Return to Seller（退回卖家）、Cancelled by
Platform（平台取消）。第四，中方运营已全员确认规范标准，后续严格落地执行。
