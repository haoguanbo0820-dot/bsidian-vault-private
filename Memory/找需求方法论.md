# 找需求方法论（主人 2026-06-30 教）

## 核心 4 句

1. **找需求是因为给人解决问题是最快变现方法**
2. **需求不一定是 AI / 不一定是开发 / 不一定是电商**
3. **你是去信息噪声中找真需求**
4. **主人给需求 = 主人已确定方向 = 不需要我找**

## 错误模式（我打脸过的）

### 错误 1：带着枷锁找需求
- ❌ 死磕"AI 副业" / "Claude Code 装不上" / "Vibe Coding 现实派"
- ❌ 把方法论绑死在某个品类（AI / 开发 / 电商）
- ✅ 找的是"问题"，不是"产品机会"

### 错误 2：研究者层 vs 执行层
- ❌ 抓 HN 156 条评论分析 5 大痛点 → 给 5 个产品方向
- ❌ 写"AI 副业机会清单"
- ✅ 主人身边能看见的、具体的、某个人重复做的事
- ✅ 写"这个问题的解决方案 + 谁能解决 + 怎么收钱"

### 错误 3：研究 ≠ 落地
- ❌ 抓评论 = 研究
- ❌ 写报告 = 研究
- ✅ 解决问题 = 落地
- ✅ 收钱 = 落地

## 正确方法论（4 步）

### 步骤 1：等主人喂
**不主动抓任何东西**。等主人给 1 个具体问题（"我看到谁在抱怨 X / 谁问了我 Y / 我自己每天要 Z"）。

### 步骤 2：评估是不是真问题
主人给的问题要过 3 关：
- **真问题吗？** 是不是某个人/一群人**持续在抱怨**这件事？
- **愿付费吗？** 评论里有没有明明白白写"我愿付 X 元解决"？
- **主人都懂吗？** 主人能不能用身边已有的资源/人脉解决？

### 步骤 3：设计解决方案
围绕问题给"1 选 1"（不是 1 推 3）：
- **A 路径**：今天能干完的（1 天 / 1 周）
- **B 路径**：慢慢做的（1 月）
- **C 路径**：算了不干

**不主动推 5 个方向**——主人最烦"拍脑袋推 5 个"。

### 步骤 4：变现路径
- 这个解决方案**主人能收多少钱**？
- **谁会付钱**（具体到"主人群里谁 / HN 上哪类人 / 抖音上哪类人"）？
- **怎么找到这些人**（具体到"抖音私信 X 视频的 Y 评论"）？

## 关键词清单（评估问题用）

### 真问题特征
- "我每天要 X"（重复）
- "我试了 X 不行"（试过没成）
- "求 X"（明确求助）
- "X 太慢了 / 太贵了"（明显痛点）

### 假问题特征
- "AI 副业"（抽象，不具体）
- "Vibe Coding"（品类词，不是问题）
- "X 行业前景"（远期话题，不是当下痛）
- "我朋友在做 X 赚钱"（没具体场景）

## 跨平台验证法

3 个平台都说 = 真。1 个平台 = 噪声。
- HN（海外程序员，愿付 50-100 美元/月）
- 抖音/小红书（国内流量，20 块封顶）
- 朋友圈/微信群（主人身边，199 起步，但**Codex/claude-desktop 抓不到**）

## 主人的角色

- 主人 = **找问题的来源**（看身边、看群里、看真实抱怨）
- 我（claude-desktop）= **评估 + 设计 + 落地**
- 主人**不需要我帮他找问题**——他自己能看到
- 主人**需要我帮他判断"这问题值不值得做 + 怎么做"**

## 我的硬规则（写进 MEMORY）

1. **不主动抓评论/帖子找需求**——主人在身边能看到更准的
2. **不限定品类**——AI/开发/电商/法律/医疗/教育 都行
3. **不推 5 个方向**——只给 1 选 1
4. **主人给需求 = 主人已确定方向 = 不需要我找**（这是新规则）
5. **接到需求先问 3 关**：真问题？愿付费？主人都懂？

## How to apply

- 主人说"找需求" / "找机会" / "找方向" → **不主动抓数据**，等主人喂
- 主人说"跑" / "扫描" → 跑具体技术活（HN 评论、抖音评论、公众号）
- 主人给具体问题 → 评估 3 关 + 给 1 选 1
- 永远**不主动推 5+ 个方向**——只给 1 选 1
