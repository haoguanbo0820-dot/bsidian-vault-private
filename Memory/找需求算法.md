# 找需求算法（v1，主人 7/1 教）

> **核心**：
> 1. **学逻辑**（主人教）
> 2. **生成公式/算法**（把方法论变可执行）
> 3. **用算法主动找**（跑数据）
> 4. **主人确认**（给候选，主人拍板）

## 4 步流程

| 步骤 | 我做什么 | 主人做什么 |
|---|---|---|
| 1. 学逻辑 | 读 vault/听主人教/记到 MEMORY | 教 |
| 2. 生成算法 | 写公式/规则/启发式 | review |
| 3. 跑算法 | 抓数据/过滤/打分 | — |
| 4. 给候选 | 排序后给"1 选 1" + 证据 | **确认**（pick / reject / 调整）|

## 找需求算法 v2（**主人 7/1 10:15 修正**）

### 输入
- 主人 vault 业务上下文（CEL 30 万单/天，中俄跨境物流）
- 抓取源：**三维数据**（公开平台 + 行业数据 + 主人身边**补充**）
- 关键词：不限品类（不预设 AI/开发/电商）

### Step 1：抓热（多源 + 三维）
抓多个平台的热榜/主题/搜索结果。
- **公开平台**: HN / 抖音 / 小红书 / 闲鱼 / 知乎
- **行业数据**: 商务部 / 国家统计局 / 海关总署 / 雨果跨境
- **主人身边**（**补充**，**不是终极**）: 朋友圈 / 微信群 / 老板群

**输出**：raw 列表（去重）

### Step 2：过滤假声音
- 过滤营销标题/标题党/重复/已发过
- **保留条件**：
  - 评论数 > 50
  - 或互动 > 1000
  - 或销量 > 100
  - 或点赞 > 5000

**输出**：候选列表（去重 + 量化）

### Step 3：挖评论
- 抓 top 100 评论
- **找"愿付费"信号**：
  - "我愿付 X 元"
  - "求装/求做/求帮忙"
  - "公司让我自费"
  - "求 X 教程/资源"
- **过滤噪音**：
  - "👍""学习了""马克" = 噪音
  - 评论里没数字 = 假

**输出**：候选评论 + 信号强度

### Step 4：跨平台验证（**三维**）
- **公开平台**: HN/抖音/小红书/闲鱼 **3 个都提 = 真**
- **行业数据**: 海关/商务部/统计局 数据支持 = **加分**
- **主人身边**: 朋友圈/微信群 提到 = **加分**（**不是唯一**）

**输出**：真需求候选（带证据链 + 三维评分）

### Step 5：排序 + 1 选 1
**排序维度**：
- 真问题（3 平台验证）> 假问题
- 愿付费 > 不愿
- 主人都懂 > 不懂
- 三维全有 > 二维 > 一维
- 1 选 1 形式给

**输出格式**：
```
候选 N: [问题名]
- 信号: [3 平台评论 + 行业数据 + 主人身边]
- 谁付钱: [具体到职业/群]
- 主人能做: [1 句话]
- 1 选 1: [现在干 / 等 / 算了]
- 证据: [链接 + 评论截图 + 数据图表]
```

## 关键：主人确认什么？

**不是定方向**——方向我（算法）提。
**是拍板**：
- ✅ "这个对，我现在就干"
- ❌ "这个假，跳过"
- 🔄 "这个方向对但不是这个具体问题，再找"

## 错误模式（我之前犯的）

| 错 | 对 |
|---|---|
| ❌ "等主人喂" = 偷懒 | ✅ 主动跑算法给候选 |
| ❌ 给 5 个方向让主人选 | ✅ 给 1-3 个候选 + 证据 |
| ❌ 主人给问题我做执行 | ✅ 我提候选主人拍板 |
| ❌ "研究层"分析 | ✅ "执行层" 5 步算法 |
| ❌ 不限定品类=借口不抓 | ✅ 不限定品类=更广找 |
| ❌ "主人身边 = 终极源" | ✅ **三维数据：公开 + 行业 + 主人身边** |
