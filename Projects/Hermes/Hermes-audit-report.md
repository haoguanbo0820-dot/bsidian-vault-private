## 三、严重问题

### [CRITICAL] 1. API Key 硬编码在源码中
**文件**: `src/analyzer.py:10`
MiniMax API Key 以明文硬编码。风险: 代码泄露 → API Key 被盗 → 产生费用。

### [CRITICAL] 2. 无依赖清单
**文件**: 项目根目录
无 `requirements.txt`、`setup.py` 或 `pyproject.toml`。7 个文件使用 `sys.path.insert(0, ...)` 导入模块。

### [CRITICAL] 3. 数据库密码明文传递
**文件**: `sync_engine.py:37` → `load_passphrase()`
DPAPI 解密后的 hex 密码在 6 个模块间以字符串传递，可能被日志/core dump 泄露。

### [HIGH] 4. 核心同步链路零测试覆盖
`sync_room_messages()`、`run_sync()`、`LocalDB` 全部无测试。旧测试引用已不存在的类/方法。

### [HIGH] 5. Web UI 非生产就绪
Flask 开发服务器（单线程、无 WSGI），配置页存在闭包捕获隐患。

### [HIGH] 6. Web UI 配置页缺少引擎-模型联动
选择引擎后模型列表不自动更新。
