# 物流群聊知识库系统设计

> 2026-06-25 | 基于微信群聊自动构建物流业务知识库
> 来源: `wechat-daily/docs/superpowers/specs/`

---

## 概述

从日常群聊中提取报价、货物参数、运输方案、问题跟进、业务决策等信息，与预置的市场基准数据 + 外部事件对照，实现利润空间可视化和知识沉淀。

---

## 核心业务模型

### 客户关心的三个核心问题

```
1. 运费 ── 多少钱？
2. 时效 ── 几天到？
3. 安全到达 ── 货会不会坏、会不会丢？
```

物流不可能三角：

| 方案 | 运费 | 时效 | 安全 | 适合场景 |
|------|:---:|:---:|:---:|---------|
| 空运 | 贵 | 快(3-5天) | 中转多易损 | 高价值/紧急补货 |
| 铁路 | 中 | 中(12-25天) | 较安全 | 大宗/B2B/机械设备 |
| 卡航 | 中 | 中(7-12天) | 全程可视 | 中小批量/季节性 |
| 海运 | 低 | 慢(30-50天) | 较安全 | 非紧急大宗 |

### 运输模式矩阵

```
国内运输:
├── 快递 (三通一达/顺丰/京东/跨越)
│   └── 首重+续重, 月结大客户协议价
├── 同城配送 → 距离+重量+件数, 阶梯计价
├── 大件物流 → 按件/按方/按吨
├── 零担拼车 (LTL) → 按方或按吨分摊
├── 整车 (FTL) → 按车型一口价包车(4.2m~17.5m)
└── 专线 → 按线路固定报价

国际运输 (中俄线):
├── 铁路 (中欧班列/车皮/单柜, 12-25天)
├── 海运 FCL (20GP/40GP/40HQ)
├── 海运 LCL (拼箱按方)
├── 卡航 (汽运7-12天)
├── 空运 (3-5天)
├── 双清包税 / 白关清关
└── 口岸: 满洲里/绥芬河/霍尔果斯/阿拉山口
```

### 计价逻辑

| 模式 | 计价依据 | 转换关系 |
|------|---------|---------|
| 快递 | 首重+续重 | 体积重=长×宽×高÷6000, 实重与体积重择大 |
| 零担/专线 | 按方/按吨 | 3立方米≈1吨, 轻货按方重货按吨 |
| 整车 | 按车型一口价 | 含油费+路桥+人工+利润 |
| 回程车 | 市场价5-7折 | 看方向, 热门线路更低 |
| 海运FCL | 按柜型 | 含海运费+附加费 |
| 铁路 | 按柜/按吨 | 含拖车+报关+班列费+口岸费 |

### 报价构成

```
基础运费
+ 提货费（上门 vs 送货到仓）
+ 报关费 / 清关费
+ THC / DOC / 订舱费（海运）
+ 过磅费 / 量尺费
+ 进仓费 / 卸货费
+ 派送费（门口 vs 上楼）
+ 保险费（千分之一到千分之三）
+ 查验费
+ 燃油附加费、旺季附加费、货币附加费...
```

同样的价格，包含的服务不同，实际利润完全不同。

### 三层价格体系

```
市场价 (运满满/货车帮行情) ── 外部参考基准
    ↓ 对比
合约价 (车队/线路的长期协议底价) ── 实际成本
    ↓ 加利润
报价 (报给客户的成交价) ── 从聊天提取
```

系统自动计算：利润率 = (报价 - 合约价) / 报价

### 第四维度：外部环境

国际物流受外部事件冲击极大：

| 事件类型 | 实例 | 影响 |
|---------|------|------|
| 战争/制裁 | 俄乌战争 | 保险+$0.5/kg, 部分银行拒付 |
| 清关政策突变 | EAC认证收紧 | 认证周期+30天, 成本+$1/kg |
| 税务变更 | VAT 20%→22% | 税负+2% |
| 海关执法 | 低报严查 | 必须如实申报, 部分品类停运 |
| 汇率波动 | 卢布/人民币汇率 | 直接影响利润 |

## 数据库设计（9 张新表）

1. **vehicle_types** — 车型基准库
2. **price_references** — 市场行情参考
3. **contract_prices** — 合约价/资源价
4. **logistics_objects** — 物流对象 (quote/cargo/shipment)
5. **logistics_attributes** — 对象属性 (KV存储，灵活扩展)
6. **external_events** — 外部事件（战争/制裁/政策/汇率）
7. **issues** — 问题追踪库
8. **issue_updates** — 问题跟进时间线
9. **decisions** — 决策沉淀

### 预定义属性键

- 运费维度: `price`, `price_unit`, `price_type`, `inclusions`, `exclusions`, `payment_terms`
- 时效维度: `transit_days`, `etd`, `eta`, `route`, `transport_mode`
- 安全维度: `packaging`, `is_fragile`, `can_stack`, `cargo_type`, `insurance`
- 货物维度: `length_cm`, `width_cm`, `height_cm`, `weight_kg`, `cargo_cbm`

## AI Prompt 增强

在现有日报 prompt 基础上新增物流业务提取指令：

- **logistics_objects**: 从群聊提取报价/货物/运输方案（含运输模式、路线、车型、尺寸、重量、包装、价格）
- **price_comparison**: 三层价格对比（市场价 vs 合约价 vs 报价）
- **decisions**: 关键决策（谁、什么决策、标签、上下文）

## 外部事件采集

### 监测关键词池

```
清关类:   俄罗斯清关新政 / EAC认证 / 报关要求变更 / 禁运清单
税务类:   俄罗斯增值税 / 关税调整
运输类:   中俄班列 / 铁路运价 / 口岸拥堵
制裁类:   对俄制裁 / 银行限制 / 保险变更
汇率类:   卢布对人民币 / 美元对卢布
平台类:   Ozon政策 / WB要求 / 跨境新规
```

### 采集流程

```
每天定时任务 → WebSearch 搜索关键词 → AI 提取新增事件 → 写入 external_events (confirmed=0) → 日报提示待确认
```
